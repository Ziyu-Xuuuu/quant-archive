# 市场状态识别系统使用指南

## 一、系统概述

本系统是一个基于 HMM（隐马尔可夫模型）的量化交易框架，主要功能包括：

1. **趋势结构识别**（T1~T4）：识别市场的趋势状态
2. **波动率结构识别**（V1~V4）：识别市场的波动特征
3. **路径依赖状态识别**（S1~S5）：识别市场所处的交易阶段
4. **HMM 状态学习**：通过无监督学习自动发现市场隐藏状态

## 二、核心文件说明

### 2.1 主要脚本

- **[regime_hmm.py](regime_hmm.py)** - 核心状态识别引擎
  - 包含所有技术指标计算函数
  - 实现趋势/波动/路径依赖的分类逻辑
  - 提供 HMM 模型训练和预测功能

- **[run_state_on_601788_csv.py](run_state_on_601788_csv.py)** - 运行脚本示例
  - 演示如何对单个股票数据运行状态识别
  - 输入：历史行情 CSV 文件
  - 输出：带状态标签的 CSV 文件

- **[main.py](main.py)** - 回测主程序
  - 整合多种交易策略
  - 使用 MetaModelStrategy 进行模型组合
  - 支持多种 selection_mode 对比测试

### 2.2 策略文件

- **[strategies/meta_model_strategy.py](strategies/meta_model_strategy.py)** - 元模型策略
  - 集成 LSTM、HMM、Transformer 三种预测模型
  - 根据市场状态动态选择最优模型
  - 支持四种组合模式：`three_models`、`lstm_hmm`、`lstm_transformer`、`hmm_transformer`

## 三、状态分类体系

### 3.1 趋势结构（Trend Regime）

| 状态 | 含义 | 特征 |
|------|------|------|
| **T1** | 上升趋势 | 多周期均线多头排列 + ADX > 25 + RSI > 55 + MACD 金叉 |
| **T2** | 低波动震荡 | ADX < 20 + RSI 中性（45~55）|
| **T3** | 下降趋势 | 多周期均线空头排列 + ADX > 25 + RSI < 45 + MACD 死叉 |
| **T4** | 高波动/反转 | 中等 ADX + 布林带突破 + 高波动 |
| **UNK** | 未知状态 | 不满足以上条件，留给 HMM 学习 |

### 3.2 波动率结构（Volatility Regime）

| 状态 | 含义 | 特征 |
|------|------|------|
| **V1** | 低波动收敛 | GARCH 波动率低 + 斜率 ≤ 0 + 布林带窄 + RSI 中性 |
| **V2** | 中波动稳定 | GARCH 波动率中等 + 斜率平稳 + RSI 中性 |
| **V3** | 高波动发散 | GARCH 波动率高 + 斜率 > 0 + 布林带宽 + RSI 极端 |
| **V4** | 高波动衰减 | GARCH 波动率高但斜率 < 0 + RSI 回归中性 |

### 3.3 路径依赖状态（State）

| 状态 | 含义 | 转换条件 |
|------|------|----------|
| **S1** | 趋势启动期 | 从 S5/S4 进入 T1/T3，ADX 从低位上升 |
| **S2** | 趋势延续期 | 保持 T1/T3，ADX 高位稳定 |
| **S3** | 趋势衰竭期 | T1/T3 但 ADX 回落 + 高波动（V3/V4）|
| **S4** | 高波动震荡 | T2/T4 + 高波动（V3/V4）|
| **S5** | 恢复/修复期 | 震荡 + 波动回落，等待下一轮趋势 |

## 四、常见问题及解决方案

### 问题 1：UNK 状态过多

**现象**：`trend_regime` 列中 UNK 占比超过 50%

**原因**：
- 规则条件过于严格，大部分数据点不满足 T1~T4 的判定条件
- 技术指标计算窗口期导致前期数据缺失

**解决方案**：

1. **调整阈值参数**（在 [regime_hmm.py:361-379](regime_hmm.py#L361-L379)）：
```python
TREND_PARAMS_DEFAULT = {
    "adx_weak": 20.0,      # 降低到 15.0 可放宽条件
    "adx_strong": 25.0,    # 降低到 20.0 可放宽条件
    "rsi_mid_low": 45.0,   # 调整到 40.0 扩大中性区间
    "rsi_mid_high": 55.0,  # 调整到 60.0 扩大中性区间
    # ... 其他参数
}
```

2. **使用 HMM 填补 UNK**：
   - HMM 会自动学习所有数据点的隐状态
   - 通过 `hmm_state_label` 列查看 HMM 对 UNK 的解释

3. **调试工具**：运行 `debug_trend_conditions()` 查看各条件命中率
```python
debug_trend_conditions(df_feat)
```

### 问题 2：HMM 状态分布不均

**现象**：5 个 HMM 状态中某些状态占比过高或过低

**原因**：
- 数据样本不足（< 200 条）
- 特征选择不当
- HMM 超参数需要调整

**解决方案**：

1. **增加训练样本**：确保至少有 500+ 条历史数据

2. **调整特征列**（在 [regime_hmm.py:849](regime_hmm.py#L849)）：
```python
feature_cols = ["ret", "adx", "bb_width", "vol_20", "rsi", "macd_hist"]
# 可以添加更多特征，如：
# feature_cols = ["ret", "adx", "bb_width", "vol_20", "rsi", "macd_hist", "hurst", "t_stat"]
```

3. **调整 HMM 参数**（在 [regime_hmm.py:858-863](regime_hmm.py#L858-L863)）：
```python
model_hmm = GaussianHMM(
    n_components=5,           # 可以尝试 3~7 个状态
    covariance_type="full",   # 可选 "diag"、"spherical" 降低复杂度
    n_iter=100,               # 增加到 200 提高收敛性
    random_state=42,
)
```

### 问题 3：状态转换过于频繁

**现象**：相邻时间点的状态频繁切换，缺乏稳定性

**解决方案**：

1. **增加平滑窗口**：
```python
# 对状态序列做众数平滑
df_feat["state_smoothed"] = df_feat["state"].rolling(window=5, min_periods=1).apply(
    lambda x: x.mode()[0] if len(x.mode()) > 0 else x.iloc[-1]

```

2. **添加状态持续时间约束**：
```python

min_duration = 3
# 实现逻辑见后续代码示例
```

### 问题 4：回测中模型预测失败

**现象**：运行 [main.py](main.py) 时出现 NaN 或模型加载错误

**原因**：
- 模型文件路径不正确
- 数据窗口不足 lookback 长度
- 特征列缺失或包含 NaN

**解决方案**：

1. **检查模型文件**：

2. **检查数据完整性**：

3. **调整 lookback 参数**：

**回测流程**：
1. **数据准备** → 加载历史行情数据
2. **状态识别**（可选）→ 使用 `regime_hmm.py` 生成市场状态
3. **策略初始化** → 创建策略对象
4. **信号生成** → 策略根据数据生成交易信号
5. **资金曲线计算** → 回测引擎计算收益
6. **结果可视化** → 绘制资金曲线和指标

### 9.2 创建新的交易策略

#### 步骤 1：继承 BaseStrategy 类

在 `strategies/` 目录下创建新文件，例如 `my_new_strategy.py`：

```python
# strategies/my_new_strategy.py
import pandas as pd
from strategies.base_strategy import BaseStrategy

class MyNewStrategy(BaseStrategy):
    def __init__(self, param1=10, param2=20):
        """
        初始化策略参数
        """
        self.param1 = param1
        self.param2 = param2

    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        """
        生成交易信号

        参数:
            df: 包含 OHLCV 数据的 DataFrame
                必需列: open, high, low, close, vol
                可选列: meta_predicted_close (如果使用预测模型)
                       state, trend_regime, vol_regime (如果使用状态识别)

        返回:
            pd.Series: 交易信号序列
                - 0.0: 空仓
                - 1.0: 满仓
                - 0.5: 半仓（可选）
        """
        df = df.copy()

        # 1. 计算技术指标
        df['indicator1'] = df['close'].rolling(window=self.param1).mean()
        df['indicator2'] = df['close'].rolling(window=self.param2).mean()

        # 2. 生成信号逻辑
        signal = pd.Series(0.0, index=df.index)
        signal[df['indicator1'] > df['indicator2']] = 1.0

        # 3. 处理 NaN 值
        signal = signal.fillna(0.0)

        return signal
```

```python
class StateAwareStrategy(BaseStrategy):
    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        df = df.copy()
        signal = pd.Series(0.0, index=df.index)

        # 检查是否包含状态列
        if 'state' not in df.columns:
            print("警告: 数据中没有 state 列，使用默认逻辑")
            # 使用默认策略逻辑
            return self._default_logic(df)

        # 根据市场状态调整策略
        # S1: 趋势启动期 - 激进建仓
        signal[df['state'] == 'S1'] = 1.0

        # S2: 趋势延续期 - 持有满仓
        signal[df['state'] == 'S2'] = 1.0

        # S3: 趋势衰竭期 - 减仓
        signal[df['state'] == 'S3'] = 0.5

        # S4: 高波动震荡 - 空仓观望
        signal[df['state'] == 'S4'] = 0.0

        # S5: 恢复期 - 轻仓试探
        signal[df['state'] == 'S5'] = 0.3

        return signal
```

#### 方法 2：在策略内部实时计算状态

```python
from regime_hmm import compute_states_from_df

class DynamicStateStrategy(BaseStrategy):
    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        df = df.copy()

        # 确保有 date 列
        if 'date' not in df.columns:
            df = df.reset_index()
            df = df.rename(columns={df.columns[0]: 'date'})

        # 实时计算市场状态
        df_with_states = compute_states_from_df(df)

        # 基于状态生成信号
        signal = pd.Series(0.0, index=df_with_states.index)

        # 趋势状态逻辑
        is_uptrend = df_with_states['trend_regime'] == 'T1'
        is_downtrend = df_with_states['trend_regime'] == 'T3'

        # 波动率状态逻辑
        is_low_vol = df_with_states['vol_regime'] == 'V1'

        # 组合条件：上升趋势 + 低波动 = 满仓
        signal[(is_uptrend) & (is_low_vol)] = 1.0

        return signal
```

### 9.4 集成预测模型

#### 使用 MetaModel 的预测结果

回测引擎会自动将 `MetaModelStrategy` 的预测结果存储到 `df['meta_predicted_close']` 列中，其他策略可以直接使用：

```python
class PredictionBasedStrategy(BaseStrategy):
    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        df = df.copy()
        signal = pd.Series(0.0, index=df.index)

        # 检查是否有预测列
        if 'meta_predicted_close' in df.columns:
            # 使用预测值计算未来收益率
            df['predicted_return'] = df['meta_predicted_close'].pct_change().shift(-1)

            # 如果预测上涨 > 2%，则买入
            signal[df['predicted_return'] > 0.02] = 1.0

            # 如果预测下跌 > 2%，则卖出
            signal[df['predicted_return'] < -0.02] = 0.0
        else:
            # 回退到基于真实价格的逻辑
            df['actual_return'] = df['close'].pct_change()
            signal[df['actual_return'] > 0] = 1.0

        return signal.fillna(0.0)
```

### 9.5 完整示例：状态驱动的动态策略

```python
# strategies/state_driven_strategy.py
import pandas as pd
from strategies.base_strategy import BaseStrategy

class StateDrivenStrategy(BaseStrategy):
    """
    根据市场状态动态调整交易逻辑的策略
    """
    def __init__(self, use_prediction=True):
        self.use_prediction = use_prediction

    def generate_signals(self, df: pd.DataFrame) -> pd.Series:
        df = df.copy()
        signal = pd.Series(0.0, index=df.index)

        # 选择使用预测价格还是真实价格
        price_col = 'meta_predicted_close' if (
            self.use_prediction and 'meta_predicted_close' in df.columns
        ) else 'close'

        # 计算技术指标
        df['ma_fast'] = df[price_col].rolling(5).mean()
        df['ma_slow'] = df[price_col].rolling(20).mean()
        df['rsi'] = self._compute_rsi(df[price_col], 14)

        # 如果有状态信息，使用状态驱动逻辑
        if 'state' in df.columns:
            for i in df.index:
                state = df.loc[i, 'state']

                if state == 'S1':  # 趋势启动期
                    # 金叉且 RSI 未超买 -> 满仓
                    if df.loc[i, 'ma_fast'] > df.loc[i, 'ma_slow'] and df.loc[i, 'rsi'] < 70:
                        signal[i] = 1.0

                elif state == 'S2':  # 趋势延续期
                    # 保持趋势方向
                    if df.loc[i, 'ma_fast'] > df.loc[i, 'ma_slow']:
                        signal[i] = 1.0

                elif state == 'S3':  # 趋势衰竭期
                    # 减仓，但不完全退出
                    if df.loc[i, 'ma_fast'] > df.loc[i, 'ma_slow']:
                        signal[i] = 0.5

                elif state == 'S4':  # 高波动震荡
                    # 空仓观望
                    signal[i] = 0.0

                elif state == 'S5':  # 恢复期
                    # RSI 超卖时轻仓试探
                    if df.loc[i, 'rsi'] < 30:
                        signal[i] = 0.3
        else:
            # 无状态信息时使用简单均线策略
            signal[df['ma_fast'] > df['ma_slow']] = 1.0

        return signal.fillna(0.0)

    def _compute_rsi(self, prices, period=14):
        delta = prices.diff()
        gain = (delta.where(delta > 0, 0)).rolling(window=period).mean()
        loss = (-delta.where(delta < 0, 0)).rolling(window=period).mean()
        rs = gain / loss
        rsi = 100 - (100 / (1 + rs))
        return rsi
```
